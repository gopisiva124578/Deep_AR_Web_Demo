Main class for interacting with DeepAR engine.

Constructor

DeepAR(params)

Used to initialize the DeepAR SDK.

IMPORTANT: Note that this is called without the new keyword.

List of initialization parameters:
  • canvasWidth : width of the canvas surface where the camera feed with AR objects will be rendered
  • canvasHeight : height of the canvas surface where the camera feed with AR objects will be rendered
  • canvas : canvas DOM object
  • numberOfFaces : how many faces the SDK will simultaneously track, the maximum is 4 and the more faces it tracks, the performance becomes more computationally expensive
  • libPath : relative path to the lib folder, the folder contents must be the same as the lib folder from the SDK download
  • segmentationInfoZip : there is a segmentation.zip file inside SDK lib folder, one must provide this path if you are going to use background segmentation features of web SDK
List of callback functions:
  • onInitialize() : the method is called when the engine is initialized
  • onVideoStarted() : called when the video is started
  • onAnimationTransitionedToState(newState) : called when the animation player transitions to a new state, the newState is the name of the state
  • onScreenshotTaken(photo) : called when the screenshot is taken, the photo parameter is the result of HTMLCanvasElement.toDataURL("image/png")
  • onScreenshotTakenCanvas(canvas) : called when the screenshot is taken, the canvas parameter is an HTMLCanvasElement
  • onCameraPermissionAsked() : called when the camera permission is asked
  • onCameraPermissionDenied() : called when the camera permission is denied
  • onCameraPermissionGranted() : called when the camera permission is granted
  • onImageVisibilityChanged() : called whenever the image enters or exits the camera field of view
  • onFaceVisibilityChanged(visible) : called whenever the face enters or exits the camera field of view
  • onFaceTracked(faceData) : called for each camera frame with the face data
  • onError(errorType, message) : called when any DeepAR related event happens
  • onNumberOfFacesVisibleChanged(numberOfFaces) : called when the number of detected faces changes
Parameters:
Name Type Description
params object An object containing the DeepAR initialization parameters. See example below.
Example
var deepAR = DeepAR({
  canvasWidth: window.innerWidth, 
  canvasHeight: window.innerHeight,
  canvas: document.getElementById('deepar-canvas'),
  numberOfFaces: 1,
  libPath: './lib',
  segmentationInfoZip: 'segmentation.zip', // relative to the lib path
  onInitialize: function() {
    // called when the initialization is finished
  }
});

});

setFaceDetectionSensitivity(sensitivity)

This method allows the user to change face detection sensitivity.
Parameters:
Name Type Description
sensitivity int The sensitivity parameter can range from 0 to 3, where 0 is the fastest but might not recognize smaller (further away) faces, and 3 is the slowest but will find smaller faces. By default, this parameter is set to 1.

setVideoElement(videoElement, mirror)

Used to pass the HTMLVideoElement to the DeepAR SDK. The SDK will grab frames from the video element and render the frames with masks/filters to the canvas. This method should be used instead of startVideo when you want to handle getUserMedia outside of the SDK or you need to apply the masks to any video stream.
Parameters:
Name Type Description
videoElement HTMLVideoElement Video DOM element from where the SDK will grab the frames.
mirror bool Mirror the video horizontally.

switchEffect(face, slot, path, callback)

Downloads the effect file and loads it into the engine. The method used to switch any effect in the scene.
Parameters:
Name Type Description
face int Specifies on which face the effect will be applied. The allowed values are 0, 1, 2 and 3.
slot string Specifies a namespace for the effect in the scene. In each slot, there can be only one effect. If you load another effect in the same slot the previous one will be removed and replaced with a new effect.
path string Path to the effect file exported from DeepAR studio.
callback function Callback to be called when the switching is complete.

switchEffectByteArray(face, slot, byteArray)

Switches the effect in the scene to the in-memory loaded effect (from the byte array).
Parameters:
Name Type Description

slot string Specifies a namespace for the effect in the scene. In each slot, there can be only one effect. If you load another effect in the same slot the previous one will be removed and replaced with a new effect.
byteArray Array.<byte> The DeepAR studio effect data.
